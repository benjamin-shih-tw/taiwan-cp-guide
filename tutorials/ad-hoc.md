# Ad-hoc 雜題與通靈引導

**簡介**
何謂 Ad-hoc，在競程圈講白話就是「沒有套路，全靠通靈跟智商」的題目。
這種題目你背了線段樹、Dijkstra 還是動態規劃都沒用。只要看穿背後的規律，程式碼通常不到十行；沒看穿的話，刻幾百行複雜的資料結構最後也是必定炸開 TLE。

---

## 解鎖 Ad-hoc 的思維工具箱

* **小數據手推**：N 很大會炸？先乖乖拿紙筆推 N = 1, 2, 3, 4, 5，看有沒有週期、對稱性或費氏數列之類的規律。
* **找不變量**：題目一直叫你操作變換？抓出那些 **絕對不會變的東西**（例如總和餘數、奇偶性、逆序對的奇偶性）。
* **看極端值**：直接看最大值、最小值，或是圖形最左邊、最上面的點。通常極端元素的限制最多，最好突破。
* **逆向思考**：正著推分支太多？試著從「目標狀態」倒推回「初始狀態」，往往能大幅減少分支。
* **分類討論**：把大問題切成幾個互斥的小狀況（奇數與偶數、正數與負數），逐一擊破。

## 經典範例：Boyer-Moore 多數投票演算法

一樣，我們先不要碰演算法，先用例子建立直覺。

如果用一般的寫法會怎麼寫？

### 暴力解

就直接雙重迴圈去算每個數字出現幾次。
複雜度：$O(N^2)$，一定炸。

### 雜湊表 (Hash Map) 解

開一個 Map 統計頻率。
複雜度：時間 $O(N)$ 很棒，但是空間也是 $O(N)$，直接違反題目規則出局。

### 排序解

先跑一次 `sort` 然後抓中間那個 `v[N/2]`。
複雜度：$O(N \log N)$，算有點慢了，而且面試官或題目要的是 $O(N)$。

---

### 通靈解 (Boyer-Moore)

這就是 Ad-hoc 巧妙的地方。核心思想是 **消除法**。

怎麼寫 寫寫看？
你只要維護兩個變數：`candidate` (目前的候選人) 跟 `count` (淨票數)。

* 遇到一樣的數字：票數 +1。
* 遇到不一樣的數字：票數 -1（互毆抵消）。
* 票數歸零：代表前面的人都死光了，換當前這個數字當新的候選人。

**code**

不考慮自己寫嗎…？為了競程實戰，這邊直接給你最精簡的 C++ 模板（包含防禦性的二次驗證）：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    // 優化標準 I/O 速度，CP 必備
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    if (!(cin >> n)) return 0;
    
    vector<int> v(n);
    for (int i = 0; i < n; i++) cin >> v[i];

    int candidate = 0, count = 0;

    // 第一階段：Boyer-Moore 大亂鬥抵消
    for (int x : v) {
        if (count == 0) {
            candidate = x;
            count = 1;
        } else if (x == candidate) {
            count++;
        } else {
            count--;
        }
    }

    // 第二階段：驗證該候選人是否真的過半
    int actual_count = 0;
    for (int x : v) {
        if (x == candidate) actual_count++;
    }

    if (actual_count > n / 2) {
        cout << candidate << "\n";
    } else {
        cout << -1 << "\n";
    }
    
    return 0;
}

```

複雜度：
時間 $O(N)$，陣列掃兩次而已，不會炸。
空間 $O(1)$，完美。

**result**
AC 啦，哪次不 AC。

## 四個常見錯誤

寫 Ad-hoc 最容易踩的雷，一次列給你：

| 錯誤 | 後果 | 怎麼避免 |
| --- | --- | --- |
| **忘記做第二階段驗證** | 送出直接 WA | 只要題目沒說「保證有多數元素」，算完一定要再掃一次確認真實次數 |
| **整數溢位 (Overflow)** | WA 找半天 | 看到累積總和、乘法，不要省空間，請直接開 `long long` |
| **極端值沒處理** | RE 或 WA | 程式碼開頭先加上 `if(n == 1) return v[0];` 這種特判防禦 |
| **陣列越界** | RE (Runtime Error) | 寫 `v[i+1]` 操作變換的時候，隨時注意 `i+1 < n` 的邊界 |

## 練習題

從簡單到難，去通靈吧：

**必做題 (照順序)**

* Codeforces Div.2 A/B/C 題 ⇒ 練手速跟基礎觀察，通常不需要高深的演算法。
* AtCoder Beginner Contest (ABC) C/D/E 題 ⇒ 日式競程特愛數學跟構造題，訓練邏輯與不變量的絕佳靶場。

**進階 (有空再做)**

* 構造題 (Constructive Algorithms) ⇒ 試著找出對稱性或從極端值反推，構造出特定的矩陣或陣列。
* 博弈題 (Game Theory) ⇒ 觀察先手與後手在小數據下的必勝態與必敗態，進而歸納出奇偶性。
