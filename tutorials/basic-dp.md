> 通常許多子問題非常相似，為此動態規劃法試圖僅僅解決每個子問題一次，從而減少計算量：一旦某個給定子問題的解已經算出，則將其記憶化儲存，以便下次需要同一個子問題解之時直接查表。這種做法在重複子問題的數目關於輸入的規模呈指數增長時特別有用。

### dp 腦袋解

我們運用 **BRUTE FORCE 解 的 “核心思想”** 也就是推每一格的數字但是推完以後我們直接一波 **記憶化（就像想起來學校到家一樣）**
我們把所有的值全部存在一個陣列裡面 每次只要需要用它我就查看就好了 這樣的時間複雜度和空間複雜度皆為 $O(n)$
具體做法如下

**code**

```cpp
#include <bits/stdc++.h>
using namespace std;
int main() {
    vector<long long> dp(82,0);
    dp[1] = 1;
    dp[2] = 2;
    
    for(int i=3; i <= 81; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    int n;
    while(cin >> n && n) {
        if (n == 0) continue;
        cout << dp[n] << '\n';
    }
    return 0;
}

```

## 遇到題目以後. . . ?

這邊講解 dp problem solving SOP
首先你要先確認那是一題 DP 題目(?
而不是 greedy 或其他有的沒的

### 為啥？

⇒ 因為 通常 greedy 的實作比較快又比較簡單好寫，如果你想成用 dp 解會浪費許多時間
那又要怎麼確認是一題 DP 題目呢
聽各種大佬的說法就是你刷提到一定程度以後就可以自動判斷
但是也是有一些 理性的判斷法
那基本上 dp 題目就是三大步

### 1. 定義狀態＋定義轉移式

#### a. 定義狀態

這邊又有兩條路可以走

**使用函式 dp**
也就是使用函式回傳的方式去求值
通常長這樣：

```cpp
int dp(int x,int y,int z){
	// code
	return re;
}

```

之類的 反正這邊只要意會到有這方法就好

**陣列 dp**
也就是顧名思義使用陣列去處理 dp 轉移
如同【經典例題】的解法一樣
通常長這樣：

```cpp
int dp[size][size]...// 題目會根據維度需求會有不同定義
// 或是
vector<int> dp(size)...

```

#### b. 轉移式

也就是你如何靠你有記憶的部分去推算沒有記憶的部分
如同【經典例題】的解法
轉移式會長這樣：

```cpp
dp[i] = dp[i-1] + dp[i-2]; // 之類的

```

關鍵在於要使用 **以前有的資訊** 的怎麼個 **組合或是計算方法** 來得出 **現在的值**
這邊可能會偏抽象 這邊也是整個 dp 題目通最 cost 時間的部分

那這邊要注意一件事 也就是說如果沒有定義好 **最最最一開始的值** 那後面不管再怎麼對 怎麼快 都沒用 因為呢你連從 [學校] 走回 [家] 都不會就別想回家了
那這個“初始值”通常稱為 **邊界條件** 每一題也會有不同的邊界條件 這也是自己要好好想的部分

那這邊又要引入偏難理解的概念 但是請一定要搞懂兩者的差異 因為後面會 **瘋狂用到**

### Top down 與 Bottom up

**Top down** 是從上往下推
你呼叫 $F(n)$ 發現你根本沒有 $F(n-1)$ , $F(n-2)$
就去算  $F(n-1) …$ 你會發現他是一個往下走了過程 所以叫 Top down

```cpp
const int MOD = 1000000007;
 
int cal_dp(int n) {
    if (n == 1) return 1;
    if (n == 2) return 1;
    if (visited[n] == 1) return dp[n];
    visited[n] = 1;
    dp[n] = (cal_dp(n - 1) + cal_dp(n - 2)) % MOD;
    return dp[n];
}

```

**Bottom up** 顧名思義就是希望「從底下算上來」
那就是說我用前面的資訊慢慢推出來現在要的東西
因此，Bottom up 的核心就是「試圖從最基礎的 case 一路算上來」

```cpp
const int MOD = 1000000007;
 
int cal_dp(int n) {
    dp[1] = dp[2] = 1;
    for (int i = 3; i <= n; ++i)
        dp[i] = (dp[i - 1] + dp[i - 2]) % MOD;
    return dp[n];
}

```

結束這部分後面就都簡單了（吧

### 2. 遞推（轉移）

這邊就是根據你的 **轉移式** 還有你最一開始存的 **邊界條件**
去慢慢推算你的答案 這部分也 **通常** 時間複雜度是 $O(n)$

### 3. 輸出答案

那輸出答案就是把你存的不管是怎麼個存法的 dp 遞推的資訊 轉換成題目要的答案
應該沒有很難理解吧 譬如你存了 費氏數列 各項，啊題目要第 n 項你就輸出你 n 項 👍
